## 背景

在v3→v3.1的迭代过程中，我们引入了Thread、reply_to_msg_id、execution_context等新结构。在这个过程中产生了一个核心疑问：设计越来越复杂，AI能用得上这些内容吗？

这个问题比架构本身更重要，因为它触及了一个根本性的设计原则。

---

## 核心原则

**结构服务于Prompt质量，不服务于模型完备性。**

---

## AI看不到数据结构

AI agent的“大脑”是一个LLM。每次决策，它收到的就是一段拼好的文本prompt。它不知道什么是Thread，不知道什么是reply_to_msg_id，不知道什么是execution_context。它看到的是：

```javascript
你是技术主管，负责用户注册功能。
这是最近的讨论：
  dev_1说：后端用JWT还是Session？
  dev_2说：JWT更适合前后端分离。
现在dev_1发来消息说：API接口定义好了，请审阅。
请决定下一步。
```

Thread、reply_to、execution_context——这些全是**PromptBuilder内部用来组装这段文本的工具**。AI不需要理解它们，AI只需要拿到一段好的上下文。

---

## 评判标准：加了这个结构，Prompt质量变好了吗？

每个数据结构的增加都应该用这个标准审视：

| 结构 | 对Prompt质量的影响 | 结论 |
|---|---|---|
| Thread | 有。PromptBuilder可以区分“当前在讨论什么”和“之前别的话题”，热上下文更精准 | 保留 |
| reply_to_msg_id | 有，但优先级不高。单聊和少人数讨论时间序就够用，并行群聊时才真正需要 | 保留，但可选，不急 |
| execution_context | 弱。初期agent在本地目录干活，不需要正式的资源管理 | Phase 0-2不加 |
| Gate | 有。门禁条件直接写进prompt，AI知道“什么算完成” | 保留 |
| DashboardData | 无直接影响 | 需要时再加 |
| Decision.retrieval_trace | 弱。调试用，不影响prompt质量 | 需要时再加 |

---

## 实践启示

### 关于复杂度

数据结构的复杂度和AI能力之间没有直接关系。AI不会因为你的数据库schema更复杂而变得更聪明。它变得更聪明的唯一原因是每次决策时拿到的prompt更准确、更相关、噪声更少。

### 关于AI是否“知道”这些关系

AI不需要知道底层的数据关系。但AI需要知道业务关系——“这个任务是谁派给我的”、“我有哪些子任务”、“当前在讨论什么”。这些信息通过PromptBuilder从数据结构里提取出来，用自然语言写进prompt里。

**数据结构是给系统看的，prompt是给AI看的。**

### 关于添加新结构的决策流程

每次要加新的数据结构时，先问两个问题：
1. **这个结构加了之后，PromptBuilder组装出来的prompt会明显变好吗？** 如果是，加。如果只是“模型更完备”但prompt没变化，不加。
2. **现在加还是以后加？** 如果当前Phase还用不到，先不加。接口已经预留了扩展点，以后加不会动骨架。

## Phase 0-2真正需要的模型

经过这次反思，Phase 0-2的最小模型集：

| 模型 | 作用 | 必要性 |
|---|---|---|
| Task | 骨架、状态机、子任务树 | 核心，必须 |
| TaskStep | 唯一事实来源 | 核心，必须 |
| Message | 消息通信 | 核心，必须 |
| Thread | 会话分组，提升热上下文精度 | 核心，必须 |
| Decision | 决策记录+版本追溯 | 核心，必须 |
| Artifact | 交付物 | 核心，必须 |
| Agent | 智能体注册 | 核心，必须 |
| Gate | 门禁规则 | Phase 2加 |
| Pattern | 模式模板 | Phase 3加 |
| Schedule | 定时任务 | Phase 2加 |
| DashboardData | 看板数据区 | Phase 3加 |
| execution_context | Task的执行环境 | Phase 2加 |

---

## 与v3/v3.1的关系

这不是推翻v3.1的设计，而是为v3.1的设计添加一个审视标准。v3.1定义的所有结构都是正确的，但不是所有结构都需要从第一天就实现。接口已经预留了扩展点，每个结构在它真正能提升Prompt质量的时候再加入，不会动骨架。

**核心准则：不要为了结构的完备性而增加复杂度，要为了prompt的质量而增加结构。**
