# Feature 模板说明

本目录提供 feature 文档模板，按顺序使用：

1. `requirement.md`：业务需求、边界、验收口径
2. `prd.md`：阶段拆解、交付计划、风险
3. `tech-spec.md`：实现设计、契约、测试、发布回滚

使用方式：

1. 创建目录：`docs/features/<feature-id>/`
2. 复制三份模板到新目录
3. 填写 `IssueRef`、`SpecRef`、Owner、状态
4. 审批通过后将文档状态从 `Draft` 改为 `Active`

命名规范见：`docs/standards/naming-and-ids.md`
